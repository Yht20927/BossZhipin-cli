# BOSS 直聘平台使用说明

> 本文档记录了使用 boss-cli 时需要注意的平台行为特点和最佳实践。
> boss-cli 通过用户浏览器中安装的油猴脚本，利用浏览器原生 API 执行操作。
> 本文不涉及任何逆向工程、破解或安全机制绕过的内容。

## 1. boss-cli 的安全模型

```
你的终端 (CLI)  →  Bridge Server (localhost)  →  你的浏览器 (油猴脚本)  →  BOSS 直聘
                                                      ↑
                                         使用浏览器原生 fetch()
                                         自动携带你已有的合法登录态
```

**为什么是安全的：**
- CLI 只和本机 Bridge Server 通信，不经过外部服务器
- 油猴脚本在浏览器中执行，使用浏览器原生的 `fetch()` API
- 所有请求由你已登录的浏览器发出，BOSS 看到的是正常用户请求
- 写操作强制节流约 48s，接近人工操作节奏

## 2. 会话 Token 与 code:37

BOSS 直聘 API 请求需要有效的会话 Token。当 Token 过期时返回 `code:37`，这是正常现象。

**处理方式：**

```bash
node cli.js refresh
```

该命令导航浏览器到 BOSS 职位页，触发平台自动刷新会话，然后重连 Bridge。整个过程约 5-10 秒。

---

```bash
node cli.js token info   # 查看缓存状态
```

## 3. API 错误码速查

| code | 含义 | 操作 |
|---|---|---|
| `0` | 成功 | — |
| `37` | 会话过期 | `node cli.js refresh` |
| `1` | 参数错误 | 检查命令参数 |
| `100` | 未登录 | 浏览器中登录 BOSS |

## 4. 操作频率限制

boss-cli 内置保护机制：

| 操作 | 限制 | 类型 |
|---|---|---|
| 搜索 | 建议 ≥ 2s 间隔 | 可通过 `--interval` 调整 |
| 打招呼 | **硬性最小约 48s** | Server 端强制，不可绕过 |
| 查看详情 | 无强制限制 | 读操作 |

**重要：** 写操作的 48s 间隔是 server 端硬编码保护，确保操作频率安全。请勿尝试修改代码绕过——这既是保护你的账号，也是遵守平台规范。

## 5. 安全使用建议

1. 先用 `--dry-run` 预览，确认无误再实际发送
2. 单日打招呼建议不超过 50 条
3. 仅用于个人求职，不用于商业用途

## 6. 技术实现

- **Bridge Server**：本地 localhost 通信桥梁 (`127.0.0.1:19425`)
- **油猴脚本**：Tampermonkey 扩展注入，使用浏览器原生 `fetch()` 调用 BOSS 公开 API
- **登录态复用**：利用浏览器已有 Cookie，无需管理密码
- **通信方式**：WebSocket 优先，HTTP 轮询兜底

本方案**不涉及**对 BOSS 直聘代码的修改、逆向或破解，不伪造浏览器环境，不使用自动化浏览器操控工具。
